# Department Tooling Plan

## Principles
- tools must be department-scoped
- all tool use must be logged
- deterministic first, probabilistic only where justified
- no unrestricted shell, browser, or internet access
- no cross-department memory leakage

## Planned Tool Categories
- OCR
- artifact persistence
- reporting
- structured web search
- supplier lookup
- pricing lookup
- release lookup
- vector memory
- embedding search

## Department Guidance
- Ash: approval, reporting, synthesis
- Brock/Misty: read-only advisory
- Gary: artifact creation, marketing outputs, reporting
- Giovanni: finance data, reporting, artifact persistence
- Professor Oak: structured research, artifact persistence, release/sourcing interpretation
- Bill: runtime diagnostics, workflow files, artifact persistence
- Todd/Tracey: writing, refinement, artifacts

## Tool Expansion Order
1. OCR and artifact persistence
2. structured search for finance/research only
3. vector memory for scoped domains
4. embeddings for workflow retrieval
5. supplier/pricing/release lookup with approvals

## Blocked Tool Classes
- unrestricted shell
- autonomous browser control
- open-ended internet actions
- blanket memory access
- uncontrolled delegation
